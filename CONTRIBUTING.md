# Contributing

Thanks for your interest in improving this project.

## What to contribute

Good contributions include:
- better public feed examples
- safer or cleaner n8n node structure
- alternate delivery channels like Slack or email
- improved scoring/filtering rules
- clearer demo documentation
- additional public-safe industry variants

## Contribution rules

- Keep the project public-demo-safe
- Do not add private credentials, real client data, or internal URLs
- Prefer mock/sample data for demos
- Keep workflow examples easy to explain and easy to import

## Local checklist

Before opening a PR:
- validate JSON files
- open `app/index.html` locally and check layout
- confirm the README still matches the workflow
- confirm no secrets or client references were introduced

## Suggested PR format

Please include:
- what changed
- why it improves the project
- whether it changes the portfolio/demo story
- screenshots if the public-facing page changed
