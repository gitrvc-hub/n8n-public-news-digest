# Screenshot / Preview Asset Checklist

Use this list before publishing the repo publicly.

## Recommended assets

### 1. Hero screenshot
Capture the top section of:
- `app/index.html`

This becomes:
- GitHub README image
- portfolio thumbnail
- social preview image

### 2. n8n canvas screenshot
After importing the workflow, take one clean screenshot showing:
- Schedule Trigger
- feed fetch/parsing steps
- scoring
- AI summarization
- delivery node

### 3. Digest output screenshot
Capture a sample digest in:
- Telegram
- Slack
- or email

Use only sample/public-safe output.

## Naming convention

Save images under `assets/` using names like:
- `assets/hero-preview.png`
- `assets/n8n-canvas.png`
- `assets/digest-output.png`

## Recording a Loom

Use `docs/demo-script.md`.

Recommended structure:
1. 5 sec — what the project is
2. 15 sec — show the public-facing page
3. 20 sec — show the workflow logic in n8n
4. 15 sec — show sample data and output
5. 5 sec — explain business value

## Publishing note

Do not block publishing if screenshots are not ready yet.
The repo is already structured correctly. These assets are just the polish layer.
